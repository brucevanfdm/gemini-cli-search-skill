# 高级检索策略：多轮 Gemini + 精准 Fetch

对于深度调研任务，推荐采用**分层递进式**多轮检索策略，结合 Gemini 的推理能力和 Fetch 的精准获取。

## 策略流程

```
第一轮：Gemini Brain 广泛搜索 → 获取概览、关键词、行业分析
    ↓
第二轮：Gemini Brain 深入特定主题 → 详情、攻击场景、工具检测逻辑
    ↓
第三轮：web_fetch 精准获取 → 官方文档、GitHub 项目详情
    ↓
整合：交叉验证、生成结构化报告
```

## 适用场景

| 场景 | 单轮 Gemini | 多轮 + Fetch |
|------|-------------|--------------|
| 简单事实查询 | 足够 | 过度设计 |
| 深度安全调研 | 信息浅 | CVE 编号 + 攻击流程 |
| 技术选型对比 | 概述 | 具体版本 + 性能数据 |
| 漏洞分析 | 通用建议 | 具体漏洞 + 防御代码 |

## 实战示例：安全漏洞调研

**第一轮 - 广泛搜索**：
```bash
gemini-cli-search "深度调研 OpenClaw AI 助手的安全架构、已知漏洞、官方安全建议" --search
```

**第二轮 - 深入特定主题**：
```bash
gemini-cli-search "深入研究 CVE-2026-25253 漏洞详情、供应链风险、MCP 协议攻击场景" --search
```

**第三轮 - 精准获取**：

从第二轮 Gemini 输出中提取具体 URL（官方文档、GitHub 项目页、CVE 详情页等），然后用 `WebFetch` 获取原文：

```bash
# 示例：获取 Gemini 搜索结果中提到的具体链接
WebFetch "<从 Gemini 输出中提取的实际 URL>"
```

> 注意：不要凭空构造 URL。只 fetch Gemini 搜索结果中明确给出的链接。

## 策略优势

| 优势 | 说明 |
|------|------|
| **深度挖掘** | Gemini 发现关联信息，Fetch 获取准确原文 |
| **交叉验证** | 多轮结果相互印证，减少幻觉 |
| **结构化** | 每轮聚焦特定维度，输出更有条理 |
| **可追溯** | 每轮都有明确的引用和来源 |

## 注意事项

1. **配额管理**：Gemini 免费额度 60 请求/分钟，多轮检索需注意配额分配
2. **失败处理**：某一轮失败时，可降级为 web_search 补充
3. **时效性**：Fetch 的文档可能更新，建议标注获取时间
